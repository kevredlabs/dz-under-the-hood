Le Problème : l'internet public ne peut pas faire le #3

Internet Public — les faits
  • BGP optimise la disponibilité, pas la vitesse
  • 10-15 sauts FAI par paire de validateurs
  • Jitter imprévisible (50–200 ms)
  • Chaque validateur déduplique seul (×1000)
→
Ce que vous subissez en tant que dev
  • Slots ratés → blockhash expiré → BlockhashNotFound
  • Logique de retry complexe dans vos apps
  • MEV asymétrique : vos users se font front-runner
  • Données RPC en retard → état onchain périmé

Qu'est-ce que DoubleZero ?

L'internet public, c'est une route nationale partagée avec tout le monde.
DoubleZero, c'est une autoroute privée réservée aux validateurs Solana —
plus courte, sans détours, sans embouteillages.

  • Réseau fibre privé décentralisé — overlay sur des fibres sous-utilisées
  • Des contributeurs fournissent bande passante + matériel, récompensés en 2Z
  • Validateurs et nœuds RPC s'y connectent via des points d'entrée dédiés
  • Optimise le trafic critique : blocs, shreds, votes de consensus

Ce que ça change pour vous

Moins de transactions ratées

Slot skip rate réduit → votre blockhash reste valide plus longtemps → moins de BlockhashNotFound, moins de retries à gérer dans votre code.

Confirmation plus rapide

Shreds livrés plus vite → blocs confirmés plus tôt → finality accélérée. Vos utilisateurs ne fixent plus un spinner.

MEV plus équitable

L'équité temporelle rend le front-running aveugle structurellement plus difficile. Meilleure protection pour vos utilisateurs et vos protocoles DeFi.

Nouveaux designs possibles

Cancel-before-fill, fair sequencing, ordres limites on-chain… Des primitives financières impossibles sans jitter minimal deviennent réalisables.

DoubleZero est une infrastructure de protocole — vous n'intégrez pas de SDK. Vous en bénéficiez dès que votre RPC ou vos validateurs s'y connectent.

Sous le Capot : 3 mécanismes clés

Tunnel réseau VXLAN

Les données voyagent dans une enveloppe privée, invisible depuis l'extérieur. Personne sur l'internet public ne peut voir ni intercepter le trafic des validateurs.

Carte GPS automatique BGP

Dès qu'un validateur se connecte, tout le réseau sait comment l'atteindre en quelques secondes — sans configuration manuelle.

Voies séparées VRF

Le trafic de consensus (blocs, votes) ne partage jamais la même voie que le trafic de maintenance. Un pic de monitoring ne peut pas ralentir le consensus.

Le Chemin d'un Bloc dans le Mesh

1
Le leader produit un bloc

Découpé en shreds pour la distribution via Turbine

2
Les shreds entrent dans le mesh DZ

Via tunnel GRE (mode IBRL) — aucun saut FAI, pas d'internet public

3
Filtrage Edge (FPGA)

Dédup shreds + validation format + rate limiting — en matériel, au débit ligne

4
Routage dans le mesh fibre

2-4 sauts sur fibre dédiée, MTU 9000, routage optimisé

5
Livraison aux validateurs

Jitter minimal → shreds arrivent dans l'ordre → reconstruction rapide du bloc

6
Les votes de consensus remontent

Tower BFT dans le mesh — fenêtre de vote utilisée efficacement

Ce que DoubleZero débloque pour Solana

DoubleZero n'est pas juste une optimisation réseau — c'est un changement de ce qui est possible sur Solana.

Fair Sequencing

L'équité temporelle rend réalisable un ordonnancement des transactions basé sur l'heure d'arrivée réelle — et non sur les tips. Base d'un marché financier truly fair.

Cancel-Before-Fill

Les market makers peuvent annuler leurs ordres avant qu'ils soient exécutés — prérequis #1 de Toly pour le Nasdaq décentralisé.

Ordres Limites On-Chain

Des spreads plus serrés, des carnets d'ordres on-chain viables, une liquidité plus profonde — directement accessible à vos protocoles DeFi.

DZ en mode IBRL

  • Propagation des shreds : Leader → Root Node → gros validateurs → petits validateurs
  • La latence augmente à chaque niveau
  • DZ en IBRL classique : shreds sur fibre dédiée, validateurs les plus stakés reçoivent plus vite

IBRL vs Multicast

  • Multicast (schéma ci-dessus) : Leader → réseau DZ → envoi direct à chaque validateur
  • Un émetteur, N récepteurs et pas d’arbre de relais

DoubleZero Edge

  • DZ Edge: couche de transport parallèle · accès plus rapide aux shreds pour les traders · nouvelle source de revenus pour les validateurs
  • Multicast : le leader publie les shreds une fois vers un groupe DZ → le réseau DZ livre de façon géographiquement optimisée à tous les abonnés
  • Pour les validateurs : accès plus rapide aux shreds du leader + revenus en fournissant ces données via le réseau DZ
