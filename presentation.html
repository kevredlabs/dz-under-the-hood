Network Architecture

DZD — DoubleZero Device

Physical Arista switch at each end of a fiber link. Runs the DZ Agent software.

DZX — DoubleZero Exchange

Interconnection hub where different contributors' links meet. Major metros.

Edge

Connects validators + RPC to the network

Transit

Backbone only — pure routing between DZDs

Hybrid

Edge + Transit combined

WAN Link

Between 2 DZDs from the same contributor

DZX Link

Between DZDs from different contributors, at the DZX

Building Blocks: VXLAN, BGP, VRF

VXLAN — Overlay L2

Encapsulates Ethernet frames inside UDP:4789. 24-bit VNI = 16M virtual networks. Jumbo frames MTU 9000 to absorb the ~50 bytes overhead.

BGP — Inter-DZD Routing

iBGP with private ASN 65342. When a validator connects to an Edge DZD, its route is automatically propagated across the entire mesh. IS-IS internally for fast convergence.

VRF — Traffic Isolation

Isolated routing tables on the same switch: dz-prod (blocks, shreds, votes) vs dz-mgmt (SSH, monitoring). A monitoring spike can't affect consensus.

A Block's Path Through the Mesh

1
Leader produces a block

Split into shreds for distribution via Turbine

2
Shreds enter the DZ mesh

Via GRE tunnel (IBRL mode) — no ISP hops, no public internet

3
Edge Filtering (FPGA)

Shred dedup + format validation + rate limiting — in hardware, at line-rate

4
Routed through the fiber mesh

2-4 hops on dedicated fiber, MTU 9000, optimized routing

5
Delivered to validators

Minimal jitter → shreds arrive in order → fast block reconstruction

6
Consensus votes propagate back

Tower BFT through the mesh — voting window used efficiently
